# Put your avatar on the web

Turn the `.glb` you generated into an interactive 3D avatar on your portfolio —
textured or as a wireframe "hologram". Three ways, pick by your stack.

> **Live example:** [portfolio-nine-dusky-15.vercel.app](https://portfolio-nine-dusky-15.vercel.app/)
> — Next.js + React Three Fiber, using an avatar generated by this repo.

## Which file to use

| File you generated | On the web |
|---|---|
| `avatar.glb` (textured) | realistic 3D head, orbit + auto-rotate |
| `mesh.glb` (geometry-only, from `./avatar mesh`) | wireframe / hologram look |

Both are standard glTF — they work in every option below.

---

## 1. Zero build — `<model-viewer>`

Easiest. No npm, no bundler. Copy [`model-viewer.html`](./model-viewer.html) and your
`avatar.glb` to any static host (GitHub Pages, Netlify, Vercel, S3):

```html
<script type="module"
  src="https://ajax.googleapis.com/ajax/libs/model-viewer/3.5.0/model-viewer.min.js"></script>

<model-viewer src="avatar.glb" camera-controls auto-rotate
              shadow-intensity="1" exposure="1.1" environment-image="neutral">
</model-viewer>
```

## 2. Plain three.js (textured ⇄ wireframe toggle)

Open [`three-wireframe.html`](./three-wireframe.html) — full three.js scene with a
button that flips between the textured skin and a glowing wireframe. ES-module CDN,
still no build step.

## 3. Next.js / React Three Fiber  ← your portfolio's stack

Drop-in component: [`react-three-fiber/Avatar.jsx`](./react-three-fiber/Avatar.jsx).

```bash
npm i three @react-three/fiber @react-three/drei
```

```jsx
// app/page.jsx (or any client component)
import Avatar from "@/components/Avatar";   // copy Avatar.jsx here

export default function Hero() {
  return (
    <section className="hero">
      <h1>Arockiya Ossia</h1>
      {/* textured head */}
      <Avatar src="/avatar.glb" />
      {/* or the wireframe version */}
      {/* <Avatar src="/avatar_mesh.glb" wireframe /> */}
    </section>
  );
}
```

1. Copy `Avatar.jsx` into your `components/`.
2. Put `avatar.glb` (and optionally `avatar_mesh.glb`) in `public/`.
3. Render `<Avatar />` in your hero — replaces a flat `<img>` with a live 3D head.

Props: `src`, `wireframe` (bool), `autoRotate` (bool), `height` (px).

## Tips for a snappy site

- **Keep it light:** web wants < ~150k triangles. Generate a lean mesh, or decimate
  in Blender (Decimate modifier) / `meshoptimizer`. Texture 1–2K is plenty.
- **Compress:** run the `.glb` through [gltf-transform](https://gltf.report/) —
  Draco/Meshopt geometry + WebP textures often cut size 5–10×.
  ```bash
  npx @gltf-transform/cli optimize avatar.glb avatar.opt.glb --texture-compress webp
  ```
- **Lazy-load:** mount the `<Canvas>` only when the hero is in view (IntersectionObserver)
  so it doesn't block first paint.
- **Mobile:** cap `dpr={[1, 2]}` (already set) and avoid shadows on low-end devices.

## Generate the files

```bash
./avatar single  photo.jpg -o public/avatar.glb        # textured head
./avatar mesh    photo.jpg -o public/avatar_mesh.glb   # geometry-only (wireframe)
```

That's it — your face, in 3D, on your site.
